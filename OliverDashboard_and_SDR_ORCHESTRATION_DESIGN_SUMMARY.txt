================================================================================
                        KEY DESIGN CONCEPTS
================================================================================

FOLLOW-UP SPACING (Deterministic):
  First contact: Mar 10
  Follow-up 1: Mar 14 (+4 days)
  Follow-up 2: Mar 18 (+8 days)
  Follow-up 3: Mar 24 (+14 days)
  Follow-up 4: Mar 31 (+21 days)
  Follow-up 5: Apr 9 (+30 days)
  → All relative to FIRST contact date (not last send date)

SEND WINDOWS:
  → Only Tue/Wed/Thu
  → Only 9–11 AM or 1–3 PM
  → All times in prospect's LOCAL timezone
  → If today is Mon/Fri/weekend, defer to next eligible day

CONFIDENCE SCORING:
  → 0.8–1.0: Auto-use (write to Sheet, proceed to draft)
  → 0.5–0.79: Flag for user review (wait for confirmation)
  → <0.5: Skip (don't write to Sheet)

APPROVAL GATES:
  → Every draft needs [APPROVE] / [REGEN] / [SKIP] button click
  → Never send without explicit approval
  → Never flag/use data below confidence 0.8

================================================================================
                        HOW THE PIPELINE WORKS
================================================================================

Every day at 8 AM ET (Mon-Fri):

STEP 1-2 (7 min):      Health check + sync from Google Sheet
STEP 3 (10-15 min):    Enrich missing email/timezone/company info
STEP 4 (2 min):        Load knowledge base documents
STEP 5-6 (10 min):     Generate drafts + queue for approval
STEP 7-8 (10 min):     Scan inbox + classify replies
STEP 9-12 (5 min):     Log events + metrics + alerts + write back
STEP 13 (2 min):       Prompt for pending actions

TOTAL: ~1 hour
USER TIME: 2–5 minutes (just click buttons)

================================================================================
                        DESIGN HIGHLIGHTS
================================================================================

DETERMINISTIC (No Randomness):
  ✓ Same follow-up spacing every time
  ✓ Same send windows (Tue/Wed/Thu, 9–11 AM / 1–3 PM)
  ✓ Same logic every day, no ML surprises

SAFE (Multiple Gates):
  ✓ Never send without approval ([APPROVE] button)
  ✓ Always validate data (confidence < 0.8 = flag)
  ✓ Always respect opt-outs (auto-pause sequence)
  ✓ Always flag uncertainty ("?" prefix in Notes)

TRANSPARENT (You Know What's Happening):
  ✓ Daily Telegram digest shows exactly what happened
  ✓ Real-time prompts tell you when decisions are needed
  ✓ All calculations are deterministic (no black box)

EFFICIENT (Minimal User Input):
  ✓ Draft approvals: just click [APPROVE] / [REGEN] / [SKIP]
  ✓ Reply classifications: auto-classified if confidence ≥ 0.8
  ✓ Email confirmations: click [CONFIRM] if score 0.5–0.79
  ✓ Pool alerts: add prospects when prompted

================================================================================
                    CRITICAL DESIGN DECISIONS
================================================================================

1. Fixed spacing from first contact (not from last send)
   → Predictable, no edge cases, easy to explain

2. Tue/Wed/Thu only, 9–11 AM / 1–3 PM
   → Empirically best for cold email open rates

3. Confidence ≥ 0.8 for auto-use threshold
   → Balances automation with safety

4. One approval per draft
   → Prevents hallucinated claims, ensures brand voice

5. LLM-classified replies with user gate on low confidence
   → Automates obvious replies, asks user on ambiguous ones

6. Free APIs only (Hunter, Never Bounce, Abstract)
   → No vendor lock-in, low cost
