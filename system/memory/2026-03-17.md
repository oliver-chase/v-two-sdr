# Recovery Session — 2026-03-17

## Session Summary
Recovered from critical authentication state issue where OpenClaw had lost its connection to auth system. Full recovery and state documentation completed.

## What Broke

**Issue:** OpenClaw was unable to authenticate with API providers. The agent was unable to route requests properly, causing all API calls to fail.

**Root cause:** Configuration mismatch between:
- Old proxy-based auth model (localhost:18790)
- New direct auth-profiles.json model
- Misconfigured or missing auth-profiles.json settings

---

## What Was Fixed

1. **Verified auth-profiles.json location and contents**
   - File: `~/.openclaw/agents/main/agent/auth-profiles.json`
   - Confirmed OpenRouter key is present and valid

2. **Updated authentication configuration**
   - Set active profile to: `openrouter:default`
   - Key type: `or-free` (OpenRouter free tier key)
   - Model default: `openrouter/auto`

3. **Decommissioned proxy dependency**
   - Proxy (localhost:18790) still runs but is no longer required
   - OpenClaw now reads keys directly from auth-profiles.json
   - Removed proxy requirements from OpenClaw startup flow

4. **Updated documentation**
   - Updated `system/souls/secrets-management.md` with "Current State" section
   - Appended gateway health check to `system/souls/startup-checklist.md`
